# AI in QA - Overview

AI is changing how we test software. This is my personal notes page on what I have explored so far and what I am still learning. I update this as I go.

## Tools I have used or explored

**Claude AI (Anthropic)**
My main AI tool right now. I use it to generate test cases, analyse requirements, write bug reports, and review test scripts. I also built an AI Test Case Generator project using the Claude API - it takes a feature description and returns structured test cases covering functional, negative, boundary, security, UI and integration scenarios in seconds.

**ChatGPT (OpenAI)**
I use ChatGPT for general QA tasks like brainstorming test scenarios, explaining concepts, and drafting test documentation when I need a second opinion or a different angle.

**GitHub Copilot**
Tried this for writing test automation scripts. It suggests code as you type which speeds up writing repetitive test code. Useful when writing pytest tests and API test frameworks.

**Cursor**
An AI powered code editor. Similar to GitHub Copilot but more conversational - you can ask it to write, fix, or explain code inline. I have been exploring it for automation work.

**Hermes AI Agents**
We use Hermes at Sopify.AI as part of our infrastructure. It is an AI agent system that handles automated workflows and tasks across the product pipeline.

**Make.com with AI nodes**
At Farmers District 46 I build automation workflows using Make.com. It now supports AI nodes which means you can add AI steps inside your automation flows - for example summarising data or making decisions based on content.

**Mabl**
An AI native test automation platform. Currently studying this through Mabl University. It uses AI to create, maintain and run tests with minimal manual scripting. Great for teams who want automation without heavy coding.

**Applitools**
An AI powered visual testing tool. It uses AI to compare screenshots and detect visual differences that traditional assertions miss. I explored this through Test Automation University.

## What I use AI for in QA

- Generating first draft test cases for a new feature
- Identifying gaps in requirements before testing starts
- Writing clearer bug reports faster
- Reviewing automation scripts for improvements
- Generating test data for different scenarios
- Summarising test results for stakeholders

## What AI still cannot do

AI gives you speed and coverage but it does not replace QA thinking. It does not know your system, your users, or which edge cases actually matter for your product. The best results come from using AI for the heavy lifting and then applying your own judgment to refine and prioritise.

I am still learning how to get the best out of these tools. This page will grow as I explore more.
